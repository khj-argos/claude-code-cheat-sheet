## 개요

Claude Code의 서브에이전트는 소프트웨어 개발의 다양한 측면에 대한 전문적인 전문성을 제공합니다. 모든 작업에 범용 에이전트를 사용하는 대신, 특정 사용 사례에 대해 더 나은 결과를 얻기 위해 이러한 집중된 에이전트를 활용할 수 있습니다.

### 주요 이점

- **전문화된 지식**: 각 에이전트는 도메인별 전문 지식을 보유
- **최적화된 도구**: 에이전트는 전문 분야와 가장 관련성이 높은 도구에 액세스
- **더 나은 컨텍스트**: 집중된 에이전트는 도메인과 관련된 컨텍스트 유지
- **더 빠른 결과**: 전문화된 에이전트는 해당 영역에서 더 효율적으로 작업 가능

## 모범 사례

### 올바른 서브에이전트 선택

1. **작업과 전문성 일치**: 작업과 전문 분야가 일치하는 서브에이전트 선택
2. **도구 요구 사항 고려**: 일부 서브에이전트는 설계상 제한된 도구 액세스 권한 보유
3. **컨텍스트 고려**: 전문화된 에이전트는 더 나은 집중된 컨텍스트 유지
4. **복잡한 작업에는 범용 목적 사용**: 작업이 여러 도메인에 걸쳐 있을 때 범용 목적 에이전트 사용

### 효과적인 서브에이전트 사용

1. **구체적으로 작성**: 명확하고 상세한 작업 설명 제공
2. **명확한 기대치 설정**: 서브에이전트가 제공하기를 원하는 것을 명시
3. **컨텍스트 제공**: 관련 배경 정보 포함
4. **서브에이전트 체인**: 복잡한 워크플로우를 위해 여러 서브에이전트를 순차적으로 사용

### 일반적인 패턴

```bash
# 프론트엔드 개발 워크플로우
1. 프론트엔드 UI 전문가: 컴포넌트 구축
2. 프로덕션 검증자: 프로덕션 리뷰
3. 코드 문서화 담당자: 컴포넌트 문서 생성

# 코드 리뷰 워크플로우  
1. 프로덕션 검증자: 초기 품질 검사
2. 범용 목적: 필요한 경우 복잡한 분석
3. 코드 문서화 담당자: 문서 업데이트

# 조사 및 구현
1. 범용 목적: 조사 및 계획
2. 전문화된 에이전트: 구현
3. 프로덕션 검증자: 최종 리뷰
```

## 통합 워크플로우

### 다중 에이전트 워크플로우

Claude Code를 사용하면 복잡한 워크플로우를 위해 서브에이전트를 함께 연결할 수 있습니다:

1. **순차 처리**: 에이전트를 차례대로 사용
2. **병렬 처리**: 여러 에이전트를 동시에 실행
3. **조건부 로직**: 작업 요구 사항에 따라 에이전트 선택

### 예시 워크플로우

#### 완전한 기능 개발
```
1. 범용 목적: 요구 사항 조사 및 아키텍처 계획
2. 프론트엔드 UI 전문가: 사용자 인터페이스 컴포넌트 구축  
3. 프로덕션 검증자: 코드 품질 및 보안 리뷰
4. 코드 문서화 담당자: 포괄적인 문서 생성
```

#### 코드 품질 파이프라인
```
1. 프로덕션 검증자: 자동화된 품질 검사
2. 범용 목적: 복잡한 문제 분석 (필요한 경우)
3. 프론트엔드 UI 전문가: UI 관련 수정 (해당되는 경우)
4. 코드 문서화 담당자: 문서 업데이트
```
